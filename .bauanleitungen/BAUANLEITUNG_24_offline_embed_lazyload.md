# Bauanleitung 24: Offline-Embed mit Lazy-Load-Modulen

Macht die Sätze-Wiedergabe **offline-fähig** (Tool als `file://` aus
heruntergeladenem ZIP nutzbar) und behält die volle Online-
Funktionalität (HTTP/HTTPS, GitHub Pages) bei.

## Architektur — Hybrid mit Lazy-Load

- **Online (HTTP/HTTPS)**: Tool lädt wie bisher `assets/sentences/sentences.json`
  per `fetch`, voller 450-Sätze-Korpus.
- **Offline (`file://`)**: `fetch` schlägt fehl → Tool springt in den
  **Embed-Modus** und lädt pro Sprache eine kleine JS-Datei
  `assets/sentences/embed/<lang>.js` per dynamischem `<script>`-Tag
  (das geht auch unter `file://`). Pro Sprache je ~5 Sätze mit
  Audio als `data:`-URLs.
- Bei Sprachwechsel im Offline-Modus wird die jeweilige Embed-Datei
  on-demand nachgeladen (nicht alle auf einmal).

**Embed-Plan pro Sprache:**

| Sprache | Quelle im Embed | Sätze |
|---|---|---|
| de | Thorsten (erste 5 von 50) | 5 |
| en | Common Voice (erste 5 von 100) | 5 |
| fr | Common Voice (erste 5 von 100) | 5 |
| es | Common Voice (erste 5 von 100) | 5 |
| spätere Sprachen | nur online | 0 |

Jede `<lang>.js`-Datei: ~300–400 KB (5 MP3s × ~60 KB × 1.33
Base64-Overhead). Pro Sprache eine eigene Datei — Tool lädt nur die,
die aktuell gebraucht wird. Beliebig viele weitere Sprachen können
später ohne Aufblähen der initialen Page-Load hinzugefügt werden.

## Reihenfolge

1. Build-Skript `scripts/build_embed.py`
2. Ausführen → 4 Embed-Dateien erzeugen
3. `sentences.js` erweitern (Hybrid-Lader + data-URL-Support in der Wiedergabe)
4. Hook in `applyLang` für Sprachwechsel im Offline-Modus
5. SPEC.md, CODESTRUKTUR.md
6. Akzeptanztest (online + offline)
7. Selbstprüfung

## 1. Build-Skript `scripts/build_embed.py`

Neue Datei `scripts/build_embed.py` mit folgendem Inhalt:

```python
#!/usr/bin/env python3
"""
build_embed.py — erzeugt assets/sentences/embed/<lang>.js für Offline-Nutzung.

Pro Sprache eine kleine JS-Datei mit ~5 Recordings als data:-URLs.
- de: erste 5 Thorsten-Sätze
- en/fr/es: erste 5 Common-Voice-Sätze (je verschiedene Sprecher,
  bereits in sentences.json so kuratiert)

Wird lokal ausgeführt, idealerweise nach jedem sentences.json-Update
oder MP3-Refresh.

Voraussetzung: keine. Nur Standard-Python.
"""

import base64
import json
import os
import sys

REPO_ROOT = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))
SENT_DIR  = os.path.join(REPO_ROOT, "assets", "sentences")
EMBED_DIR = os.path.join(SENT_DIR, "embed")

EMBED_PER_SPEAKER = 5

# Welche Sprache nimmt welche speaker-keys ins Embed?
PLAN = {
    "de": ["thorsten"],
    "en": ["cv-en"],
    "fr": ["cv-fr"],
    "es": ["cv-es"],
}


def encode_mp3(path):
    with open(path, "rb") as f:
        data = f.read()
    return "data:audio/mp3;base64," + base64.b64encode(data).decode("ascii")


def build_lang(lang_code, speaker_keys, corpus):
    out_speakers = {}
    for skey in speaker_keys:
        spk = corpus["speakers"].get(skey)
        if not spk:
            print(f"  WARN: speaker '{skey}' fehlt in sentences.json")
            continue
        if spk.get("lang") != lang_code:
            print(f"  WARN: speaker '{skey}' hat lang '{spk.get('lang')}', erwartet '{lang_code}'")
            continue
        recs_in = spk.get("recordings", [])[:EMBED_PER_SPEAKER]
        recs_out = []
        for r in recs_in:
            mp3_path = os.path.join(SENT_DIR, r["audio"])
            if not os.path.isfile(mp3_path):
                print(f"  WARN: missing audio {mp3_path}")
                continue
            new_r = {k: v for k, v in r.items() if k != "audio"}
            new_r["audio"] = encode_mp3(mp3_path)
            recs_out.append(new_r)
        out_speakers[skey] = {
            "lang":    spk.get("lang"),
            "label":   spk.get("label"),
            "kind":    spk.get("kind"),
            "source":  spk.get("source", ""),
            "license": spk.get("license", ""),
            "credit":  spk.get("credit", ""),
            "recordings": recs_out,
        }
    return {"speakers": out_speakers}


def main():
    os.makedirs(EMBED_DIR, exist_ok=True)
    with open(os.path.join(SENT_DIR, "sentences.json"), encoding="utf-8") as f:
        corpus = json.load(f)

    for lang, spk_keys in PLAN.items():
        print(f"[{lang}] build embed ...")
        data = build_lang(lang, spk_keys, corpus)
        n_recs = sum(len(s["recordings"]) for s in data["speakers"].values())
        if n_recs == 0:
            print(f"  SKIP: keine Recordings für {lang}")
            continue
        out_path = os.path.join(EMBED_DIR, f"{lang}.js")
        # Minified-Style: ein Aufruf, keine Whitespace-Inflation.
        js = (
            "// auto-generated by scripts/build_embed.py — do not edit\n"
            "(function(){window.SENTENCES_EMBED=window.SENTENCES_EMBED||{};"
            f"window.SENTENCES_EMBED[{json.dumps(lang)}]="
            f"{json.dumps(data, ensure_ascii=False, separators=(',',':'))};"
            "})();\n"
        )
        with open(out_path, "w", encoding="utf-8") as f:
            f.write(js)
        size_kb = os.path.getsize(out_path) / 1024
        print(f"  -> {out_path} ({size_kb:.0f} KB, {n_recs} recordings)")

    print("done.")


if __name__ == "__main__":
    main()
```

## 3. `sentences.js` erweitern

### 3a. Neue State-Variablen

### 3b. `sLoadIfNeeded` erweitern um Offline-Fallback

**Vorher** (etwa Z. 35–48):

```js
async function sLoadIfNeeded() {
  if (sLoaded || sLoading) return;
  sLoading = true;
  try {
    const res = await fetch("assets/sentences/sentences.json", { cache: "no-cache" });
    if (!res.ok) throw new Error("HTTP " + res.status);
    sCorpus = await res.json();
    sLoaded = true;
  } catch (err) {
    console.error("[sentences] konnte sentences.json nicht laden:", err);
    sCorpus = { speakers: {} };
    sLoaded = true;
  } finally {
    sLoading = false;
    sUpdateUI();
  }
}
```

**Nachher** — bei fetch-Fehler in Offline-Mode wechseln und Embed der
aktuellen Sprache laden:

```js
async function sLoadIfNeeded() {
  if (sLoaded || sLoading) return;
  sLoading = true;
  try {
    const res = await fetch("assets/sentences/sentences.json", { cache: "no-cache" });
    if (!res.ok) throw new Error("HTTP " + res.status);
    sCorpus = await res.json();
    sLoaded = true;
    sOfflineMode = false;
  } catch (err) {
    console.warn("[sentences] online fetch fehlgeschlagen, wechsle in Embed-Modus:", err);
    sOfflineMode = true;
    sCorpus = { speakers: {} };
    sLoaded = true;
    // Aktuelle Sprache als Embed nachladen
    if (typeof lang !== "undefined") {
      await sEnsureEmbedForLang(lang);
    }
  } finally {
    sLoading = false;
    sUpdateUI();
  }
}

// Lädt assets/sentences/embed/<lang>.js per <script>-Tag (file://-kompatibel),
// merged die Sprecher in sCorpus. No-op wenn online oder schon geladen.
async function sEnsureEmbedForLang(langCode) {
  if (!sOfflineMode) return;
  if (!langCode) return;
  if (sEmbedLoading.has(langCode)) return;
  // Schon geladen? Wenn irgendein Sprecher in der Sprache existiert: ja.
  for (const k in sCorpus.speakers) {
    if (sCorpus.speakers[k].lang === langCode) return;
  }
  sEmbedLoading.add(langCode);
  try {
    await new Promise((resolve, reject) => {
      const s = document.createElement("script");
      s.src = "assets/sentences/embed/" + langCode + ".js";
      s.onload = () => resolve();
      s.onerror = () => reject(new Error("embed " + langCode + " nicht ladbar"));
      document.head.appendChild(s);
    });
    const data = window.SENTENCES_EMBED && window.SENTENCES_EMBED[langCode];
    if (data && data.speakers) {
      for (const k in data.speakers) {
        sCorpus.speakers[k] = data.speakers[k];
      }
    }
  } catch (err) {
    console.warn("[sentences] kein Embed für", langCode, ":", err.message);
  } finally {
    sEmbedLoading.delete(langCode);
    sUpdateUI();
  }
}
```

### 3c. `sPlayCurrent` mit data:-URL-Support

Die Wiedergabe-Funktion holt aktuell das MP3 per `fetch`. Bei Embed
sind die Audio-Pfade aber `data:audio/mp3;base64,...` — und `fetch`
auf eine data-URL klappt nicht zuverlässig (manche Browser ja, manche
nein). Sauberer Weg: data-URL direkt in ArrayBuffer wandeln.

**Vorher** (Z. 94–100):

```js
async function sPlayCurrent() {
  if (!sActive || !sCurRec) return;
  const url = "assets/sentences/" + sCurRec.rec.audio;
  try {
    const res = await fetch(url);
    if (!res.ok) throw new Error("HTTP " + res.status);
    const arrayBuf = await res.arrayBuffer();
    const c = gPC();
    const decoded = await c.decodeAudioData(arrayBuf);
    ...
```

**Nachher** — Audio-Quelle abhängig vom Pfad-Prefix:

```js
function sDataUrlToArrayBuffer(url) {
  // "data:audio/mp3;base64,XXXX" -> ArrayBuffer
  const comma = url.indexOf(",");
  const b64 = url.substring(comma + 1);
  const bin = atob(b64);
  const len = bin.length;
  const bytes = new Uint8Array(len);
  for (let i = 0; i < len; i++) bytes[i] = bin.charCodeAt(i);
  return bytes.buffer;
}

async function sPlayCurrent() {
  if (!sActive || !sCurRec) return;
  const audioRef = sCurRec.rec.audio;
  try {
    let arrayBuf;
    if (audioRef.startsWith("data:")) {
      arrayBuf = sDataUrlToArrayBuffer(audioRef);
    } else {
      const res = await fetch("assets/sentences/" + audioRef);
      if (!res.ok) throw new Error("HTTP " + res.status);
      arrayBuf = await res.arrayBuffer();
    }
    const c = gPC();
    const decoded = await c.decodeAudioData(arrayBuf);
    ...
```

Der Rest der Funktion (ab `if (!sActive) return;`) bleibt unverändert.

### 3d. `sUpdateUI` erweitert: bei Offline-Mode Embed der aktuellen Sprache nachladen

Bei Sprachwechsel im Offline-Modus muss die neue Sprache geladen
werden. Aktuell ruft `applyLang()` in i18n.js am Ende `sUpdateUI()` —
wir hängen den Embed-Loader davor.

**In `sUpdateUI()` als allerersten Block** (gleich nach `if (typeof
lang === "undefined") return;` und der card-Existenz-Prüfung) ergänzen:

```js
function sUpdateUI() {
  if (typeof lang === "undefined") return;
  const card = document.getElementById("plSentencesCard");
  if (!card) return;

  // Im Offline-Mode bei Sprachwechsel ggf. Embed nachladen.
  // (fire-and-forget; sUpdateUI wird beim Embed-Load-Ende erneut gerufen.)
  if (sOfflineMode && sLoaded) {
    sEnsureEmbedForLang(lang);
  }

  const ctrls = document.getElementById("plSentControls");
  ... (Rest unverändert)
```

## 4. Kein expliziter Hook in i18n nötig

Da `applyLang()` schon `sUpdateUI()` aufruft (aus Bauanleitung 21) und
wir den Embed-Loader jetzt in `sUpdateUI` einhängen, ist der
Sprachwechsel automatisch abgedeckt. Sonnet braucht dort nichts zu
ändern.

**Offline-Modus**: Wenn `fetch("assets/sentences/sentences.json")`

## 6. Akzeptanztest

### Online-Modus (HTTP-Server lokal)

```bash
cd ~/git/ci-sound-balancing
python3 -m http.server 8000
# Browser: http://localhost:8000
```

1. Player-Tab, Sprache Deutsch → Sprecher-Dropdown zeigt "Alle / Thorsten / Common Voice".
2. Endlosfolge starten → Sätze laufen mit voller Vielfalt (Pool 150 DE-Recordings).
3. Sprache wechseln auf Englisch → "Alle / Common Voice", 100 EN-Sprecher verfügbar.
4. Browser-Konsole: keine Warnung "wechsle in Embed-Modus".

### Offline-Modus (file://)

1. Browser-Tab schließen, dann `index.html` im Datei-Manager
   doppelklicken (öffnet als `file:///pfad/zu/index.html`).
2. Player-Tab, Sprache Deutsch.
3. Browser-Konsole zeigt: `[sentences] online fetch fehlgeschlagen, wechsle in Embed-Modus`.
4. Sprecher-Dropdown zeigt **nur Thorsten** plus "Alle (zufällig)" (CV-Pool fehlt offline).
5. **Spielen** → einer von 5 Thorsten-Sätzen wird abgespielt.
6. **Nächster Satz** mehrfach → wechselt zwischen den 5 Embed-Sätzen.
7. **Endlosfolge** → läuft durch die 5 Sätze in zufälliger Folge.
8. Sprache auf Englisch wechseln.
   → Konsole zeigt evtl. kurz Lade-Vorgang `embed/en.js`.
   → Sprecher-Dropdown zeigt "Alle / Common Voice" (nur cv-en aus Embed).
9. **Endlosfolge** auf Englisch → 5 verschiedene CV-Sprecher.
10. Sprache auf Französisch, Spanisch → analog.
11. Wenn du in i18n.js testweise eine Sprache ergänzt, für die es
    keinen Embed gibt (z.B. `it` würde nichts laden):
    → Sätze-Block zeigt "keine Sätze verfügbar".

### EQ / MAPLAW im Offline-Modus

12. EQ-Toggle ein, Korrektur aktiv, Endlosfolge.
    → Sätze klingen mit Korrekturkurve. Beweis, dass data-URL-Audio
    durch denselben Audiograph läuft.

## Hinweis zur Weiterentwicklung

Wenn du später eine neue Sprache hinzufügst:

1. `scripts/fetch_common_voice.py --lang <iso> --count 100` ausführen
   (im Codespace oder lokal).
2. Per Migrations-Skript aus Bauanleitung 22 (oder manuell) den neuen
   Sprecher `cv-<iso>` in `sentences.json` aufnehmen.
3. `PLAN`-Dict in `scripts/build_embed.py` um die neue Sprache
   erweitern (Eintrag `"<iso>": ["cv-<iso>"]`).
4. `python3 scripts/build_embed.py` neu laufen lassen — erzeugt die
   neue `embed/<iso>.js`.
5. Im Tool ist die Sprache dann sowohl online als auch offline
   verfügbar — ohne dass andere Sprachen Page-Load-Strafe zahlen.
